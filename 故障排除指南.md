# UI 不显示问题 - 故障排除指南

## 问题：打开的界面没有 UI

### 已修复的问题

✅ **Tailwind CSS v4 配置已更新**
- 已将 `@tailwind` 指令更新为 `@import "tailwindcss"`（Tailwind v4 的新语法）

### 检查步骤

#### 1. 清除浏览器缓存
- 按 `Cmd + Shift + R` (Mac) 或 `Ctrl + Shift + R` (Windows) 强制刷新
- 或使用无痕模式打开浏览器

#### 2. 检查开发服务器
确保开发服务器正在运行：
```bash
cd wuxing-app
npm run dev
```

#### 3. 检查浏览器控制台
打开浏览器开发者工具（F12 或 Cmd+Option+I），查看：
- **Console** 标签：是否有 JavaScript 错误
- **Network** 标签：CSS 文件是否成功加载

#### 4. 检查样式文件加载
在浏览器控制台运行：
```javascript
// 检查 Tailwind 是否加载
getComputedStyle(document.body).getPropertyValue('--tw-*')
```

#### 5. 如果仍然没有 UI

**方案 A：重启开发服务器**
```bash
# 停止当前服务器（Ctrl+C）
# 然后重新启动
npm run dev
```

**方案 B：清除并重新安装依赖**
```bash
rm -rf node_modules package-lock.json
npm install
npm run dev
```

**方案 C：降级到 Tailwind CSS v3（如果 v4 有问题）**
```bash
npm uninstall tailwindcss
npm install -D tailwindcss@^3.4.0 postcss autoprefixer
npx tailwindcss init -p
```

然后恢复 `src/index.css` 为：
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### 常见错误信息

1. **"Cannot find module 'tailwindcss'"**
   - 运行 `npm install`

2. **"PostCSS plugin tailwindcss requires PostCSS 8"**
   - Tailwind v4 需要 PostCSS 8+
   - 检查 `package.json` 中的 postcss 版本

3. **样式类名不生效**
   - 检查 `tailwind.config.js` 中的 `content` 路径是否正确
   - 确保包含所有 `.jsx` 和 `.tsx` 文件

### 验证修复

打开浏览器后，你应该看到：
1. 登录页面（如果未登录）
2. 或者主应用界面（如果已登录）

如果看到空白页面：
- 检查浏览器控制台的错误信息
- 确认 `http://localhost:5173` 可以访问
- 检查网络请求是否成功
