# 在手机上运行应用指南

## 方法一：通过局域网访问（推荐）

### 步骤 1：确保手机和电脑在同一 WiFi 网络

确保你的手机和电脑连接到同一个 WiFi 网络。

### 步骤 2：启动开发服务器

在项目目录下运行：

```bash
cd wuxing-app
npm run dev
```

### 步骤 3：查看服务器地址

启动后，终端会显示类似以下信息：

```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

### 步骤 4：在手机上访问

1. 打开手机浏览器（Safari、Chrome 等）
2. 输入 `Network` 地址（例如：`http://192.168.1.100:5173`）
3. 确保使用 `http://` 而不是 `https://`

### 获取电脑的 IP 地址

如果终端没有显示 Network 地址，可以手动查找：

**macOS/Linux:**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows:**
```bash
ipconfig
```
查找 "IPv4 地址"，通常是 `192.168.x.x` 或 `10.0.x.x`

## 方法二：使用 ngrok（跨网络访问）

如果需要从不同网络访问，可以使用 ngrok：

1. 安装 ngrok: https://ngrok.com/
2. 启动开发服务器: `npm run dev`
3. 在另一个终端运行: `ngrok http 5173`
4. 使用 ngrok 提供的 URL 在手机上访问

## 方法三：构建生产版本

如果需要长期使用，可以构建生产版本：

```bash
npm run build
```

然后使用静态文件服务器（如 `serve`）或部署到服务器。

## 常见问题

### 1. 无法访问
- 检查防火墙设置，确保 5173 端口未被阻止
- 确保手机和电脑在同一 WiFi 网络
- 尝试使用电脑的 IP 地址而不是 localhost

### 2. 页面加载慢
- 这是正常的，开发模式会实时编译
- 首次加载可能需要一些时间

### 3. 热更新不工作
- 手机浏览器可能不支持 WebSocket
- 刷新页面即可看到最新更改

## 提示

- 建议将手机浏览器添加到主屏幕，这样就像原生应用一样使用
- iOS Safari: 分享 → 添加到主屏幕
- Android Chrome: 菜单 → 添加到主屏幕
