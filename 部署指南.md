# 网页部署指南

## 方法一：使用 Vercel（推荐，最简单免费）

### 步骤：

1. **安装 Vercel CLI**（可选，也可以直接网页上传）：
   ```bash
   npm install -g vercel
   ```

2. **构建项目**：
   ```bash
   cd wuxing-app
   npm run build
   ```

3. **部署**：
   
   **方式A：使用网页界面**
   - 访问 https://vercel.com
   - 注册/登录账号（可以用GitHub账号）
   - 点击 "Add New..." → "Project"
   - 如果代码在GitHub，直接导入仓库
   - 如果没有，可以拖拽 `dist` 文件夹上传
   - 自动获得链接（如 `your-app.vercel.app`）

   **方式B：使用命令行**
   ```bash
   cd wuxing-app
   vercel
   ```
   按照提示操作即可

### 优点：
- ✅ 完全免费
- ✅ 自动HTTPS
- ✅ 全球CDN加速
- ✅ 自动部署（如果连接GitHub）
- ✅ 自定义域名支持

---

## 方法二：使用 Netlify（免费，简单）

### 步骤：

1. **构建项目**：
   ```bash
   cd wuxing-app
   npm run build
   ```

2. **部署**：
   - 访问 https://netlify.com
   - 注册/登录账号
   - 拖拽 `dist` 文件夹到部署区域
   - 自动获得链接（如 `your-app.netlify.app`）

### 优点：
- ✅ 完全免费
- ✅ 拖拽即部署
- ✅ 自动HTTPS
- ✅ 支持自定义域名

---

## 方法三：使用 GitHub Pages（免费）

### 步骤：

1. **创建GitHub仓库**：
   - 在GitHub创建新仓库
   - 将代码推送到仓库

2. **修改vite.config.js**：
   ```javascript
   export default defineConfig({
     plugins: [react()],
     base: '/your-repo-name/', // 你的仓库名
     // ... 其他配置
   })
   ```

3. **构建并部署**：
   ```bash
   npm run build
   # 将 dist 文件夹的内容推送到 gh-pages 分支
   ```

4. **启用GitHub Pages**：
   - 在仓库设置中启用GitHub Pages
   - 选择 `gh-pages` 分支或 `dist` 文件夹
   - 获得链接（如 `username.github.io/repo-name`）

---

## 方法四：使用自己的服务器

### 步骤：

1. **构建项目**：
   ```bash
   cd wuxing-app
   npm run build
   ```

2. **上传文件**：
   - 将 `dist` 文件夹中的所有文件上传到服务器
   - 配置Web服务器（Nginx/Apache）

3. **Nginx配置示例**：
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       root /path/to/dist;
       index index.html;

       location / {
           try_files $uri $uri/ /index.html;
       }
   }
   ```

## 快速部署脚本

创建一个自动化部署脚本：

```bash
#!/bin/bash
# deploy.sh

echo "开始构建..."
npm run build

echo "构建完成！"
echo "dist 文件夹已准备好，可以："
echo "1. 上传到 Vercel/Netlify"
echo "2. 上传到自己的服务器"
echo "3. 使用 GitHub Pages"
```
